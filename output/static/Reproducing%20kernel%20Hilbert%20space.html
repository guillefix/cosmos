That is, the Hilbert space has a kernel (that is a function from two copies of the input space to the reals), such that </p><ul><li>the function you get from fixing one argument of the kernel must be a member of the Hilbert space we are considering</li><li>the kernel is reproducing, that is Evaluation is done by inner producting with the kernel function with one argument fixed. see <a class="tc-tiddlylink-external" href="https://youtu.be/9-oxo_k69qs?t=1h14s" rel="noopener noreferrer" target="_blank">Theorem</a>. </li></ul><p>Note: feature vectors (see below) and functions in the RKHS are in 1-to-1 correspondence (so they can be seen to be the <em>same</em>). They can thus both be seen as the elements of the Hilbert space that is the RKHS.</p><h3 class=""><u>Positive definite kernel perspective</u></h3><p>An RKHS can be constructed given a kernel, defined to be just a function with two arguments from an input space, to the Reals, with some properties (positive definitiness). See <a class="tc-tiddlylink-external" href="https://youtu.be/e1ittn0B2iQ?t=1h5m59s" rel="noopener noreferrer" target="_blank">here</a> and <a class="tc-tiddlylink-external" href="https://youtu.be/e1ittn0B2iQ?t=1h41s" rel="noopener noreferrer" target="_blank">here</a></p><h3 class=""><u><a class="tc-tiddlylink tc-tiddlylink-missing" href="Feature%2520map.html">Feature map</a>s perspective</u></h3><p>A feature map is just a map from an input set to a <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Hilbert%2520space.html">Hilbert space</a>!</p><p><a class="tc-tiddlylink-external" href="https://youtu.be/bBRX3OqNC9c?t=19m48s" rel="noopener noreferrer" target="_blank">Video</a> – <a class="tc-tiddlylink-external" href="http://www.gatsby.ucl.ac.uk/~gretton/coursefiles/lecture4_introToRKHS.pdf#page=3" rel="noopener noreferrer" target="_blank">notes</a></p><p><strong>Idea</strong>: you map inputs to an <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Inner%2520product%2520space.html">Inner product space</a> feature space (vector of numbers, which could be a function..). products of a basis functions/vectors. So the basis functions can be seen as just the kernels, with one argument fixed. One can easily check that the inner product of two Gaussian functions (defined using the usual inner product for functions, which can be gotten by looking at functions as vectors..) is the Gaussian kernel, one can also check that this is a reproducing kernel.</p><blockquote><p>Consider a weird example of the above explanation. We can map elements of R^2 to functions on R^2 which look like pyramids centered around the input point. The space of functions has a standard inner product. Then we define a new function on R^2 given by taking the inner product of the function associated with point x with the function associated with point y for any y. Then we define the inner product of two such functions to be the inner product of the original pyramids, which makes them work like evaluating functionals. Therefore, we have defined an RKHS (the space of functions over R^2 with the inner product defined as later), via an intermediate not-necessarily-RK <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Hilbert%2520space.html">HS</a>.</p></blockquote><p><u>Connection with <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Regularization.html">Regularization</a></u></p><p>—&gt; <u>Nice</u>. The regularization term can be, for instance the norm of the derviative!</p><p>Norm of a function in the space is like a measure of &quot;<a class="tc-tiddlylink tc-tiddlylink-resolves" href="Complexity.html">Complexity</a>&quot; of that function</p><hr><p>Applied to <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Kernel%2520method.html">Kernel method</a>s</p><h3 class=""><u>Examples</u></h3><p>Space of functions in <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Dictionary%2520learning.html">Dictionary learning</a></p><p><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Band-limited%2520function.html">Band-limited function</a>s, and some relaxations of it where kernel is gaussian or exponential.. <a class="tc-tiddlylink-external" href="https://www.youtube.com/watch?v=e1ittn0B2iQ#t=37m30s" rel="noopener noreferrer" target="_blank">These generalizations are just RKHS with translational invariance kernels</a>, very common in signal analysis, etc. These are <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Sobolev%2520space.html">Sobolev space</a>s!</p><p><a class="tc-tiddlylink tc-tiddlylink-missing" href="Spline.html">Spline</a>s are also a special case! noice</p><h3 class=""><u>Properties of <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Kernel.html">Kernel</a>s</u></h3><p><a class="tc-tiddlylink-external" href="http://www.gatsby.ucl.ac.uk/~gretton/coursefiles/lecture4_introToRKHS.pdf#page=5" rel="noopener noreferrer" target="_blank">Sums and products of kernels are still kernels</a></p></div> </div> </p> </section> </body> </html>
