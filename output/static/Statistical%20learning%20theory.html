See more at <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Computational%2520learning%2520theory.html">Computational learning theory</a> and <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Probably%2520approximately%2520correct.html">Probably approximately correct</a></p><p>We have several approaches for <a class="tc-tiddlylink tc-tiddlylink-missing" href="Generalization%2520error%2520bound.html">Generalization error bound</a>s (see more detail <a class="tc-tiddlylink-external" href="https://gingkoapp.com/app#5bb786ea4176d5035eafbfb9" rel="noopener noreferrer" target="_blank">here</a>). See below.</p><p>General principle: The more that the bound depends on, the tighter it can be</p><table><tbody><tr class="evenRow"><td>Note that bounds dependent on training sample are basically bounds that depend on the <a class="tc-tiddlylink tc-tiddlylink-missing" href="Data%2520distribution.html">Data distribution</a>, but where we estimate the data-distribution-dependent bound from the sample, using some <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Concentration%2520inequality.html">Concentration inequality</a></td></tr></tbody></table><h2 class=""><u><strong>Realizable learning</strong></u></h2><p>where we assume the hypothesis class and data distribution (e.g. target function) are such that we can reach <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mn>0</mn></mrow><annotation encoding="application/x-tex">0</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.64444em;"></span><span class="strut bottom" style="height:0.64444em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathrm">0</span></span></span></span></span> <a class="tc-tiddlylink tc-tiddlylink-missing" href="Empirical%2520risk.html">Empirical risk</a>. This leads to <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Probably%2520approximately%2520correct.html">PAC learning</a> <u><strong>Data-dependent realizable learning bounds</strong></u></p><ul><li>Depends on data distribution. (Deterministc case: Depends on target function. )<ul><li>Analysis of <a class="tc-tiddlylink tc-tiddlylink-missing" href="Occam%2520algorithm.html">Occam algorithm</a>s. (for deterministic case)</li><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Rademacher%2520complexity.html">Rademacher complexity</a> analysis (average version)</li></ul></li><li>Depends on distribution over data distributions. See analysis when assuming distribution overt target functions follows the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Universal%2520probability.html">Universal probability</a> distribution. Can also do <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Bayesian%2520statistics.html">Bayesian</a> analysis in this case.</li><li>Dependent on training sample, but not on algorithm<ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Rademacher%2520complexity.html">Rademacher complexity</a> analysis. Depends on hypothesis class and training sample S</li></ul></li><li>Depends on training sample, and on algorithm<ul><li>Elementary <a class="tc-tiddlylink tc-tiddlylink-resolves" href="PAC-Bayes.html">PAC-Bayes</a> bound. Depends on the output of the learning algorithm, <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>h</mi></mrow><annotation encoding="application/x-tex">h</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.69444em;"></span><span class="strut bottom" style="height:0.69444em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">h</span></span></span></span></span>.</li><li>Realizable <a class="tc-tiddlylink tc-tiddlylink-resolves" href="PAC-Bayes.html">PAC-Bayes</a> bound. Depends on likelihood of training sample</li><li>Realizable <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Structural%2520risk%2520minimization.html">Structural risk minimization</a>. Depends on the class to which output belongs. Occam algorithms can be analyzed like this also I think.<ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Minimum%2520description%2520length.html">Minimum description length</a> is a special case.</li></ul></li></ul></li></ul><h2 class=""><u><strong><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Agnostic%2520learning.html">Agnostic learning</a></strong></u></h2><p>where we don't assume anything about the data distribution, and just ask: how well can I do relative to the function that does best (has lowest true risk) in a given hypothesis class. It has a related formulation, where we instead ask how much does the true risk differ from the empirical risk. The error in the former can be shown to be at most twice the error in the later.</p><p><u><strong>Data-dependent agnostic learning</strong></u></p><ul><li>Depends on data distribution (Deterministic case: Depends on target function.)<ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Rademacher%2520complexity.html">Rademacher complexity</a> analysis (average version))</li></ul></li><li>Depends on distribution over data distributions.</li><li>Dependent on training sample, but not on algorithm<ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Rademacher%2520complexity.html">Rademacher complexity</a> analysis. Depends on hypothesis class and training sample S</li></ul></li><li>Depends on the training sample and the algorithm<ul><li>Agnostic <a class="tc-tiddlylink tc-tiddlylink-resolves" href="PAC-Bayes.html">PAC-Bayes</a> bounds</li><li>Agnostic <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Structural%2520risk%2520minimization.html">Structural risk minimization</a>. Depends on the class to which output belongs.<ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Minimum%2520description%2520length.html">Minimum description length</a> is a special case.</li></ul></li><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Sensitivity.html">Sensitivity</a></li><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Algorithmic%2520robustness.html">Algorithmic robustness</a></li></ul></li></ul><p>(well technically, what's called &quot;realizability&quot; tends to be that the true risk is 0, while here I am using it as meaning that the empirical risk is 0)</p><hr><p><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Structural%2520risk%2520minimization.html">Structural risk minimization</a> is interesting because it only depends on the data distribution and algorithm kind of implicitly (well kinda how sample-dependent bounds depend on the data distribution implicitly.)</p></div> </div> </p> </section> </body> </html>
