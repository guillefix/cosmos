<p>See <a class="tc-tiddlylink-external" href="https://www.youtube.com/watch?v=4QgjKUzyrhM" rel="noopener noreferrer" target="_blank">here</a></p><p>For each object in a category, we can define a <strong>representable functor</strong> which is a <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Functor">Functor</a> from a category to Set, where objects are mapped to <a class="tc-tiddlylink tc-tiddlylink-missing" href="#Hom-set">Hom-set</a>s to (or from) that object, and morphisms are mapped to morphisms between arrows defined by composition. There is a covariant and a contravariant version.</p><p>There is another functor, from a category to {the <a class="tc-tiddlylink tc-tiddlylink-missing" href="#Category%20of%20functors">Category of functors</a> from the category to Set} where objects are mapped to to their corresponding <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Representable%20functor">Representable functor</a>s, and morphisms are <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Natural%20transformation">Natural transformation</a>s induced by morphisms in the original category. This is called the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Yoneda%20embedding">Yoneda embedding</a>.</p><hr><p><mark>where does the name &quot;representable&quot; come from?</mark></p>
