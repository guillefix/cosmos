process</strong> is a type of <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Explosive%2520percolation.html">Explosive percolation</a>, also known as <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.43056em;"></span><span class="strut bottom" style="height:0.43056em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">m</span></span></span></span></span>-edge processes, that involve choosing <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.43056em;"></span><span class="strut bottom" any pair of nodes (compare with other <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Spanning%2520cluster-avoiding%2520process.html">Spanning cluster-avoiding process</a>)and applying a rule to select which one is actually chosen. These have been proven to be continuous in the thermodynamic limit, for a fixed <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.43056em;"></span><span class="strut bottom" style="height:0.43056em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">m</span></span></span></span></span>. They are generalizations of Erdos-Renyi <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Random%2520graph.html">Random graph</a>s.</p><p>The first proposed type, were proposed (in <a class="tc-tiddlylink-external" href="http://science.sciencemag.org/content/323/5920/1453" rel="noopener noreferrer" target="_blank">Explosive Percolation in Random Networks</a>) and thought to maybe show a discontinuous phase transition.</p><p><u>Achlioptas processes phase transitions are continuous</u></p><p>It has now been show that the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Percolation%2520phase%2520transition.html">Percolation phase transition</a> for Achlioptas processes (and in fact a more general class of <a class="tc-tiddlylink tc-tiddlylink-resolves" href="k-vertex%2520rule%2520percolation%2520process.html">k-vertex rule percolation process</a>) is continuous (in the thermodynamic limit), but very steep (see <a class="tc-tiddlylink-external" href="http://journals.aps.org/prl/abstract/10.1103/PhysRevLett.105.255701" rel="noopener noreferrer" target="_blank">Explosive Percolation Transition is Actually Continuous</a> and <a class="tc-tiddlylink-external" href="http://projecteuclid.org/euclid.aoap/1344614200" rel="noopener noreferrer" target="_blank">Achlioptas process phase transitions are continuous</a>). One can prove the continuity by looking at the asymptotic effect of removing a single link, as the total size goes to infinity. However, Oliver Riordan and Lutz Warnke proved it by proving, in essence, that the number of subcritical components that join together to form the emergent macroscopic-sized component is not sub-extensive in system size. In the words of Friedman and Landsberg, Achlioptas Processes do not lead to the build-up of a “powder keg&quot; (which is a type of cluster configuration that does lead to discontinuous transitions).</p><p>However, the model can be generalized to one that shows genuinely discontinuous transitions (see <a class="tc-tiddlylink-external" href="http://www.nature.com/nphys/journal/v11/n7/full/nphys3378.html" rel="noopener noreferrer" target="_blank">Anomalous critical and supercritical phenomena in explosive percolation</a>). One way to achieve discontinuity is actually to allow the number of edges in the rule, <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.43056em;"></span><span class="strut bottom" style="height:0.43056em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">m</span></span></span></span></span> to scale up with <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>N</mi></mrow><annotation encoding="application/x-tex">N</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.68333em;"></span><span class="strut bottom" style="height:0.68333em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right:0.10903em;">N</span></span></span></span></span>, the network size, in a certain way.</p><p><u><span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mn>2</mn></mrow><annotation encoding="application/x-tex">2</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.64444em;"></span><span class="strut bottom" style="height:0.64444em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathrm">2</span></span></span></span></span>-edge Achli Achlioptas process</u> is the simplest type: Start with N isolated nodes and add undirected, unweighted edges one at a time. This is done by choosing, at each step, two possible edges uniformly (and independently) at random from the set of <span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>N</mi><mo>(</mo><mi>N</mi><mo>−</mo><mn>1</mn><mo>)</mo><mi mathvariant="normal">/</mi><mn>2</mn></mrow><annotation encoding="application/x-tex">N(N-1)/2</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.75em;"></span><span class="strut bottom" style="height:1em;vertical-align:-0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right:0.10903em;">N</span><span class="mopen">(</span><span class="mord mathit" style="margin-right:0.10903em;">N</span><span class="mbin">−</span><span class="mord mathrm">1</span><span class="mclose">)</span><span class="mord mathrm">/</span><span class="mord mathrm">2</span></span></span></span></span> possible {edges between a pair of distinct nodes}. One adds only one of these edges, making a choice based on a systematic rule that affects the speed of development of a GCC.</p><p><span><span class="katex"><span class="katex-mathml"><math><semantics><mrow><mi>m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height:0.43056em;"></span><span class="strut bottom" style="height:0.43056em;vertical-align:0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">m</span></span></span></span></span>-edge rules are defined similarly.</p><h2 class=""><u>Selection rules</u></h2><p><u>Product rule</u>. One choice that yields &quot;explosive&quot; percolation is to use the so-called &quot;product rule&quot;, in which one always retains the edge that minimizes the product of the sizes of the two components that it merges (with an arbitrary choice when there is a tie).</p><p><u>Sum rule</u>. The size of the new component formed is minimized.</p><p><u>Bohman–Frieze (BF) rule</u>. edge 1 is chosen if it joins two isolated vertices, and edge 2 otherwise.</p><p>A selection rule can be classified as a bounded-size or an unbounded-size rule. In a bounded-size selection rule, decisions depend only on the sizes of the components and, moreover, all sizes greater than some (rule-specific) constant <span><span class="katex"><span style="margin-right:0.07153em;">K</span></span></span></span></span> are treated identically.</p><p>There are also the more general <span><span class="katex"><span potentially more).</p><p>See more rules here: <a class="tc-tiddlylink-external" href="https://arxiv.org/abs/1405.0388" rel="noopener noreferrer" target="_blank">Explosive percolation: Unusual
