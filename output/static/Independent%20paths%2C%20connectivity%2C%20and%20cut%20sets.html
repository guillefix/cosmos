<p>Number of <strong><em>independent paths</em></strong> between two vertices (the <strong><em>connectivity</em></strong>) gives measure of how strongly connected they are. Paths can be <em>vertex-independent</em> if they share no vertex (other than starting or ending vertices), or <em>edge-independent</em> if they share no edge.</p><p>A <em>vertex (edges) <strong>cut set</strong></em> is a set of vertices (edges) that if removed will disconnect a specified pair of vertices. A <em>minimum cut set</em> is the smallest such set for the vertices (or set of vertices..). For weighted networks a minimum cut set is the set of such vertices that have the least total weight.</p><p>See more <a class="tc-tiddlylink-external" This can be shown by transforming the weighted network into a multigraph.</p><p>See more here: <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Max-flow%20min-cut%20theorem">Max-flow min-cut theorem</a>, <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Flow%20in%20a%20network">Flow in a network</a></p><h3 class="">Applications</h3><p>This result is useful because some computer algorithms (maximum flow algorithms) can compute maximum flow easily. But, by result above, they also calculate the minimum cut set size, and the connectivity, which can be used to find clusters in networks. This is in fact the current standard numerical method for connectivities and cut sets.</p><p>The max-flow/min-cut theorem has been used in a polynomial-time algorithm for finding ground states of the thermal random-field Ising model. See reference [257] in Newman's book.</p>
