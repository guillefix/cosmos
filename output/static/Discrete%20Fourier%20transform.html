<p>We approximate the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Fourier%20transform">Fourier transform</a> with a <a class="tc-tiddlylink tc-tiddlylink-missing" href="#Discrete%20transform">Discrete transform</a>. The derivation can be done with the following process:</p><p>Sample signal by multiplying by <a class="tc-tiddlylink tc-tiddlylink-missing" href="#Dirac%20comb">Dirac comb</a> –&gt; FT gets convolved with another Dirac comb, thus repeating it and making it periodic.</p><p>Then sample the FT by multiplying it with another Diract comb –&gt; then dually, the signal gets convolved by another Dirac comb which repeats it making it periodic.</p><p>As both FT and signal are now periodic, we can just look at one period of each (which now has all the information in our modified signal/FT). But because both are discretized with these Dirac combs, within one period there is now only a finite number of values. These values for the FT are the discrete Fourier transform, which can in fact be calculated easily directly from the also finite number of values within one period of the modified time-domain signal!</p><p>See <a class="tc-tiddlylink-external" href="https://users.fmrib.ox.ac.uk/~saad/ONBI/fourier_practical.html" rel="noopener noreferrer" target="_blank">Saad exercises for DTC</a></p><p>The famous <a class="tc-tiddlylink tc-tiddlylink-missing" href="#Fast%20Fourier%20transform">Fast Fourier transform</a> is just an efficient algorithm to compute the discrete Fourier transform. </p>
