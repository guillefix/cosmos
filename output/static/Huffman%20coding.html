<p>A code used in <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Data%20compression">Data compression</a> that is optimal, in the sense that it achieves the entropy limit (within less than one bit).</p><p><a class="tc-tiddlylink-external" href="https://en.wikipedia.org/wiki/Huffman_coding" rel="noopener noreferrer" target="_blank">https://en.wikipedia.org/wiki/Huffman_coding</a></p><p><a class="tc-tiddlylink-external" href="https://www.cs.cf.ac.uk/Dave/Multimedia/node210.html" rel="noopener noreferrer" target="_blank">https://www.cs.cf.ac.uk/Dave/Multimedia/node210.html</a></p><h3 class=""><u>Algorithm</u></h3><dl><dd>1. Initialization: Put all nodes in an OPEN list, keep it sorted at all times (e.g., ABCDE).</dd><dd>2. Repeat until the OPEN list has only one node left:<dl><dd>(a) From OPEN pick two nodes having the lowest frequencies/probabilities, create a parent node of them.</dd><dd>(b) Assign the sum of the children's frequencies/probabilities to the parent node and insert it into OPEN.</dd><dd>(c) Assign code 0, 1 to the two branches of the tree, and delete the children from OPEN.</dd></dl></dd></dl><p>In the animation below, the blue nodes are in the <em>OPEN list</em>., at every iteration we choose the nodes with the two lowest frequencies within the blue nodes (with preference with those not yet in the tree, if equal frequency).</p><p><img src="https://upload.wikimedia.org/wikipedia/commons/a/ac/Huffman_huff_demo.gif"></p>
