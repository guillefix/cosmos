<p><em>a.k.a. memory management</em></p><p>The <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Operating%20system">Operating system</a> allocates memory to <strong>processes</strong>, so that a process can only access that portion of memory.</p><p>This memory is divided into:</p><ul><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Call%20stack">Call stack</a>, or just the stack</li><li><a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Memory%20heap">Memory heap</a>, where dynamically allocated variables are stored.</li><li>text section.</li></ul><p>The addresses that the program uses to reference variables are actually <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Virtual%20memory">Virtual memory</a> addresses, which the operating system translates to physical memory.</p><p><a class="tc-tiddlylink-external" href="https://en.wikipedia.org/wiki/Memory_management" rel="noopener noreferrer" target="_blank">https://en.wikipedia.org/wiki/Memory_management</a></p><hr><p><a class="tc-tiddlylink-external" href="http://stackoverflow.com/questions/79923/what-and-where-are-the-stack-and-heap" rel="noopener noreferrer" target="_blank">What and where are the stack and heap?</a></p><p><a class="tc-tiddlylink-external" href="http://stackoverflow.com/questions/18446171/how-do-compilers-assign-memory-addresses-to-variables" rel="noopener noreferrer" target="_blank">http://stackoverflow.com/questions/18446171/how-do-compilers-assign-memory-addresses-to-variables</a></p>
