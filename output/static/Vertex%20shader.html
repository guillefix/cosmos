</div> <div class="tc-tiddler-body tc-reveal"><p>Takes a certain representation of a <a class="tc-tiddlylink tc-tiddlylink-missing" href="Scene.html">Scene</a>, usually as <a class="tc-tiddlylink tc-tiddlylink-missing" href="Point.html">Point</a>s (vertices), and transforms this vertices into another representation which is useful for latter processing. For instance, in <a class="tc-tiddlylink tc-tiddlylink-resolves" href="OpenGL.html">OpenGL</a>/<a class="tc-tiddlylink tc-tiddlylink-resolves" href="WebGL.html">WebGL</a>, your vertices can be transformed to what is called <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Image%2520space.html">clipspace</a>, using 4d <a class="tc-tiddlylink tc-tiddlylink-missing" href="Homogeneous%2520coordinates.html">Homogeneous coordinates</a>, which are then automatically processed to project them as 2d points in the device screen. This process is know as the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Camera%2520transform.html">Camera transform</a> (see also <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Camera%2520(computer%2520graphics).html">Camera (computer graphics)</a> for further steps)</p><p>The 2d pixel coordinates corresponding to the vertices, as well as to all points inside the polygon primitives (often triangles) that go with those vertices, are often passed to the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Fragment%2520shader.html">Fragment shader</a>, to decide which color they should appear as.</p><p>Here is a common procedure done by a vertex shader, to transform coordinates to those in clipspace: <a class="tc-tiddlylink-external" href="https://developer.mozilla.org/en-US/docs/Web/API/WebGL_API/WebGL_model_view_projection" rel="noopener noreferrer" target="_blank">WebGL model view projection</a>. </p><p>As explained <a class="tc-tiddlylink-external" href="https://stackoverflow.com/questions/7157875/in-glsl-what-is-the-formula-used-to-compute-gl-fragcoord-from-gl-position" rel="noopener noreferrer" target="_blank">here</a>, the clipspace coordinates are then transformed to pixel coordinates: gl_Position is the (4d) position of a vertex, while gl_FragCoord is the (2d) position of a fragment.</p><p>The operations that happen in between are</p><ol><li>primitive assembly (to generate a triangle from 3 vertices, e.g.)</li><li>clipping (i.e. cut the triangle in multiple triangles that are all on inside the view, if it does not initially fit)</li><li>viewport application</li><li><a class="tc-tiddlylink tc-tiddlylink-missing" href="Rasterization.html">Rasterization</a> (take those triangles and generate covered fragments from them).</li></ol><p>In WebGL, one can set the viewport, as <code>gl.viewport(0, 0, gl.canvas.width, gl.canvas.height);</code>, see <a class="tc-tiddlylink-external" href="https://webglfundamentals.org/webgl/lessons/webgl-resizing-the-canvas.html" rel="noopener noreferrer" target="_blank">here</a></p></div> </div> </p> </section>
