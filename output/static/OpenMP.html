<p><a class="tc-tiddlylink-external" href="https://www.wikiwand.com/en/OpenMP" rel="noopener noreferrer" target="_blank">Open Multi-Processing</a> is a standard found in most modern C compilers..</p><p><a class="tc-tiddlylink tc-tiddlylink-missing" href="#Parallel%20programming">Parallel programming</a> with several processors with shared memory.</p><p>One process with multiple threads of execution.</p><p>OpenMP API components:</p><ul><li>C ompiler directives </li><li>R untime library routines </li><li>E nvironment variable</li></ul><p>To create several threads for a block of code with openMP use: <code># pragma omp parallel default( none ) private( my_thread ) { /* my parallel code*/</code></p><p>The <a class="tc-tiddlylink-external" href="http://jakascorner.com/blog/2016/07/omp-default-none-and-const.html" rel="noopener noreferrer" target="_blank">default(none)</a> clause forces a programmer to explicitly specify the data-sharing attributes of all variables in a parallel region. Using this clause then forces the programmer to think about data-sharing attributes.</p><p><code>omp_get_thread_num()</code> gives a unique thread ID</p><p><code>gcc -fopenmp</code> (flag to link OpenMP runtime library)</p><p>Environment variable <code>OMP_NUM_THREADS</code> determines number of threads created by OMP</p><p><code>shared( a, b, c ) private(i)</code>: means that the variables a,b,c are on shared memory. i are on each thread's local memory</p><p><code>#pragma omp for</code>: separate for loop into threads</p><h3 class=""><u>Synchronising Threads</u></h3><p>Barrier : a given thread can only proceed once all threads have reached the barrier</p><p>Critical region : Only 1 thread can be executing codes within a critical region. </p><p>Reduction : Combining multiple private values into a single shared value</p>
