Good for media files</p><p>Really good method to recover data/files in a disk, when you know particular strings that belong to them (often for text files): <a class="tc-tiddlylink-external" href="https://superuser.com/questions/694839/recover-txt-files-based-on-known-strings" rel="noopener noreferrer" target="_blank">https://superuser.com/questions/694839/recover-txt-files-based-on-known-strings</a></p><pre><code>sudo bash -c ' for OFFSET in $(grep -Pzabio diary /dev/sda3 | cut -d: -f 1); do ((CLUSTER = OFFSET / 4096)) dd if=/dev/sda3 of=clusterCLUSTER.txtbs=4096skip=CLUSTER.txtbs=4096skip=CLUSTER count=1 done '</code></pre><p>Substitute <code>diary</code> with the search string of interest and <code>/dev/sda3</code> for the volume of interest (could be an image file). It worked great for me to recover a tiddler I accidentally overwrote. It was able to recover many versions of the file, amazingly! As long as it hasn't been overwritten in memory, this will find it.</p>
