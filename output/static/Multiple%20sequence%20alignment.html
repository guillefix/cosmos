<p>Finding the alignment of a set of sequences (which consecutive symbols match to which, while maybe allowing gaps), minimizes the number of missmatches.</p><p><a class="tc-tiddlylink-external" href="https://www.wikiwand.com/en/Multiple_sequence_alignment?wprov=sfla1" rel="noopener noreferrer" target="_blank">https://www.wikiwand.com/en/Multiple_sequence_alignment?wprov=sfla1</a></p><p>As explained here: <a class="tc-tiddlylink-external" href="https://ocw.mit.edu/courses/electrical-engineering-and-computer-science/6-096-algorithms-for-computational-biology-spring-2005/lecture-notes/lecture5_newest.pdf" rel="noopener noreferrer" target="_blank">Sequence Alignment and Dynamic Programming</a>, the problem can be solved as a minimum-distance-like <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Dynamic%20programming">Dynamic programming</a> algorithm. The main insight is expressing alignments as paths in a certain matrix.
