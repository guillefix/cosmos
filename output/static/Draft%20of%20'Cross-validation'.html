Average errors from the k iterations</dd></dl><p>More computationally expensive</p><h2 class=""><u>Leave-one-out cross-validation</u></h2><p>k-fold CV, for whem k={number of training examples}, so for each iteration, you leave one out.</p><p>Even more computationally expensive, but more accurate estimate of generalization error. Only done when the data is very scarce.</p><hr><p><em>Some thoughts</em> when I misunderstood train/validation/test learning.</p><p>What I describe here is some sort of hyperlearning where we have two steps, where we learn two sets of hyperparameters, and use two different validation sets (which I call below validation, and test, mistakenly...)</p><p>When we have trained the data using a method as hold-out CV, and with some fixed learning <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Hyperparameter.html">Hyperparameter</a>s. If we want to learn the hyperparameters, we can do this whole training procedure with several values of the hyperparameter.</p><p>However, to choose the hyperparameters that are best, we can't just look at their result at S<sub>CV</sub>, as that set has been by the algorithm to choose the final answer. As a result, the error on S<sub>CV</sub> is a biased estimator of generalization error (see <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Learning%2520theory.html">Learning theory</a>). To have an unbiased estimator, we need a test set, that is only used once the learning algorithm has finished completely. We can compare the results on the test set to choose the best set of hyperparameters. Note, that once we have done that, the test error is no longer an unbiased estimate of generalization error, as we have used it to output our very final answer; i.e. our final answer depended on it!!
