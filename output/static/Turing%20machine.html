<p>A class of abstract <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Automata%20theory">automata</a> that have the most computational power* known. It can compute anything that can be suitably represented and solved by a step by step procedure (<a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Church-Turing%20thesis">Church-Turing thesis</a>).</p><p><sub>*computational power refers to how many possible computations, i.e. sets of instructions, it can perform. See <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Theory%20of%20computation">Theory of computation</a></sub></p><p><a class="tc-tiddlylink-external" href="https://www.youtube.com/watch?v=oc0kPb-Hgvk" rel="noopener noreferrer" target="_blank">Math 574, Lesson 2-2: Turing Machines</a></p><p>The main difference between a Turing machine and a <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Finite-state%20machine">Finite-state machine</a> is that a Turing machine has unlimited memory. This is implemented by giving a Turing machine the following components:</p><ul><li>Finitely many states in a <strong>read-write head</strong>. The states form part of a <a class="tc-tiddlylink tc-tiddlylink-resolves" href="#Finite-state%20machine">finite automaton</a>, and may be called the &quot;program&quot;.</li><li>An infinite <strong>work tape</strong> on which the head can read and write. This is the &quot;memory&quot;.</li></ul><p><a class="tc-tiddlylink-external" href="https://www.youtube.com/watch?v=oc0kPb-Hgvk#t=4m35s" rel="noopener noreferrer" target="_blank">Architecture of a
