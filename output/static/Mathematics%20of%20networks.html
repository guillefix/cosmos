Edges</a></p><p><u>Paths</u></p><p>A <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Path%2520(Graph%2520theory).html">path</a> in a network is a sequence of of nodes such that every pair of nodes in the sequence is connected by an edge in the network.</p><p><a class="tc-tiddlylink-external" href="http://math.stackexchange.com/questions/655589/what-is-difference-between-cycle-path-and-circuit-in-graph-theory" rel="noopener noreferrer" target="_blank">Definition of path, cycle, trail, circuit</a> Definition is extended to directed case by only permitting traversing in the direction of edge. Note only directed graphs can have 2-cycles.</p><p><u>Components</u></p><p>A <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Component%2520(Graph%2520theory).html">component</a> is a subset of the network for which all pair of vertices have at least one path, and which is maximal (i.e no extra nodes can be added that preserve this property).</p><p><u><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Independent%2520paths%252C%2520connectivity%252C%2520and%2520cut%2520sets.html">Independent paths, connectivity, and cut sets</a></u></p><p>Number of <em>independent paths</em> between two vertices (the <em>connectivity</em>) gives measure of how strongly connected they are. Paths can be <em>vertex-independent</em> if they share no vertex (other than starting or ending vertices), or <em>edge-independent</em> if they share no edge.</p><p>A <em>vertex (edges) cut set</em> is a set of vertices (edges) that if removed will disconnect a specified pair of vertices. A <em>minimum cut set</em> is the smallest such set for the vertices.</p><p><u><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Graph%2520laplacian.html">Graph laplacian</a></u></p><p>The graph laplacian is a useful quantity, derived from the adjacency matrix, which can be used to describe diffusion precesses in a network, as well as in problems of random walks, resistor netowkrs, graph partitioning and network connectivity.</p><p><u><a class="tc-tiddlylink tc-tiddlylink-resolves" href="Random%2520walk%2520in%2520a%2520graph.html">Random walks</a></u></p><p>A <em>random walk</em> is a path across a network created by taking repeated random steps. They are usually allowed to traverse edges more than once, and visit vertices more than once. If note it is a <em>self-avoiding random walk</em>. They are mathematically connected to resistor networks.</p></div> </div> </p> </section> </body> </html>
