rel="noopener noreferrer" target="_blank">Good book</a></p><h3 class=""><u>Discrete optimizatino using energy minimization</u></h3><p>Can formulate discrete optimization as an <a class="tc-tiddlylink-external" href="http://mpawankumar.info/teaching/cdt-optimization/lecture2_2.pdf" rel="noopener noreferrer" target="_blank">Energy minimization</a> problem on a set of random variables which can take values in a discrete set. This can be formulated as a <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Graphical%2520model.html">Graphical model</a>. <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Energy-based%2520model.html">Energy-based model</a>, <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Ising%2520model.html">Ising model</a>..</p><p>In simple cases it can be solved iteratively using <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Dynamic%2520programming.html">Dynamic programming</a>. However, in general, the problem <a class="tc-tiddlylink tc-tiddlylink-resolves" href="NP-hard.html">NP-hard</a> (non-deterministic polynomial).</p><p>We can solve it (approximately?) using <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Max-flow%2520min-cut%2520theorem.html">Max-flow min-cut theorem</a>. Partition of the graph will correspond to the labelling. We will make terms in the energy correspond to capacities so that the minimum-capacity cut will correspond to minimum energy. For the energy, we will use:</p><ul><li><strong>Unary potential</strong>. Links of a particular random variable / node in the <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Graphical%2520model.html">Graphical model</a> to a source, and to a sink, with capacities corresponding to the unary potential (cost of one label or the other).</li><li><strong>Pairwise potential</strong>. Links between non-source/sink nodes. Corresponds to cost of two labels being the same (no cut through edge), or different (cut passes through edge). They may be negative, in which case we can't apply this method, and it's NP-hard! This is case is called super-modular. To apply min-cut algo, we want the weights of the link to be postivie, and this is called sub-modular. Sometimes, we can avoid supermodularity, by renaming labels, but in some cases, even this can't avoid negative links, these cases correspond to <a class="tc-tiddlylink tc-tiddlylink-resolves" href="Frustration.html">Frustration</a>. <a class="tc-tiddlylink-external" href="http://mpawankumar.info/teaching/cdt-optimization/lecture2_2.pdf" rel="noopener noreferrer" target="_blank">see here</a> (...remember you substracted P from all link weights, on slide..)</li></ul><p>Multiple labels: move-making algorithm. </p><ul><li>Expansion algorithm. <span><span class="katex"><span
